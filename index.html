Lesson One: Introduction to HTML

The Basics of the Web

There are five (5) main elements of the web:

  • the web
  • HTML (The main document type of the web)
  • URLs (How you refer to documents on the web)
  • HTTP (The protocol that unites the web; that the web runs on; that computers use to communicate with servers)
  • web applications


Vocabulary

World Wide Web: A collection of HTML documents that was invented in the early 1990's, and contains roughly 30 billion pages.

HTML : Hyper Text Mark-up Language

  • The basis for almost every web page.
  • What glues everything together.
  • The main kind of content on the web.

Hyper Links: The links that make the web web-like.

Browser:A program that runs on your computer to display files found on the web.

Internet:The world's largest computer network.

HTTP:The main protocol of the web.

Servers:Computers that host the files that make up the web.

The internet uses HTTP to communicate with you, your browser, and servers.

HTTP Request to a server. The server finds the appropriate HTML document and sends it back to the user's computer where a web browser interprets the page and displays it on the user's screen. This video does a good job of explaining.

HTML:stands for Hyper Text Mark-up Language. HTML is made up of:

  • text content (what you see)
  • markup (what the content looks like or how it's arranged)
  • references to other documents (e.g. images and videos)
  • links to other pages

This video gives a good overview.

www.udacity.com/html_playground


Tags and Elements

HTML documents are made of HTML elements. When writing HTML, we tell browsers the type of each element by using HTML tags. A void tag has no content, and therefore needs no closing tag. It is often a good idea to make a closing tag as soon as you make an opening tag. This video explains the distinction between elements and tags well.

Tag Examples :
  • b: Bold
  • em: Emphasis (italics)
  • a: Anchor (for making links)
  • img: Images (a void tag)
  • br: Break (a void tag)
  • p: Paragraph
  • u: Underline
  • ul: Unordered List
  • ol: Ordered List
  • li: List Item

Why Computers are Stupid

Computers are stupid because they interpret instructions literally. This makes them very unforgiving since a small mistake by a programmer can cause huge problems in a program.

HTML Attributes

Attributes (these days) always equal a value. Tags can have multiple attributes.

In an anchor tag, href is the attribute name, the URL is the value, and within the contents is the link to the URL.

In an image tag, src stands for the source; the URL is where the image can be found to download; alt stands for alternate; the alternate text is the text that gets displayed should the image not load. The alternate text is required for broken requests and blind people. Because there is no contents to an image, there is no closing tag.


Inline vs Block Elements

HTML elements are either inline or block. Block elements form an "invisible box" around the content inside of them.

Container Elements: Elements that do nothing but contain stuff inside them. These elements can be used to create web pages with complex structures.

  • Break (b): Inline
  • Paragraph (p): Block
  • Span: Inline (and a container element)
  • Div: Block (and a container element)

SUMMARY

  1. A web page is a text document written in a language called HTML. Web browsers read these documents, and then interpret, and display them.
  2. Coding happens when programmers write text in a language that a computer can understand. The computer can then follow the instructions the programmer wrote.
  3. Computers are stupid. Programmers need to write exactly the way a computer understands. This is also known as writing with correct "syntax".
  4. Programmers can't remember everything. There are too many details to keep everything in your head. You can always look it up.
  5. Basic HTML Vocabulary:
    • Tag: An HTML tag is always contained within angled brackets. Most tags have an opening tag and a closing tag. Void tags do not require a closing tag.
    • Element: An HTML element refers to everything within a set of opening and closing tags.
    • Attribute: This is a property of an HTML element.

Lesson Two: Creating a Structured Document

Page and Structure

Though some elements are invisible (like the "head" element), all visible elements are rectangular.

If you dig deep enough into the structure, you can read all the text that shows up on the page.

The structure of all websites containing the boxes is organized tree-like.

  • HTML and CSS are both languages.
  • HTML controls the structure of a web page.
  • CSS controls the style of a page and how it looks.
  • When programmers talk about the DOM (Document Object Model) they are talking about the tree-like structure of a page.

An element starts and ends with a tag. In its middle there can be content. This content can be text or another tag. Each tag can have attributes with values. CSS defines the style, meaning how elements look on the page. Every element is a box but with CSS you can change the form elements are displayed on the page. However, you can break down any webpage to their boxes. Boxes can be summarized with div-tags. Within these div-tags attributes can be determined that give the box a name.


Vocabulary

HTML: The language of the web. It has specific "syntax" and rules. The main word of HTML is a tag.

DOM: Document Object Model. The browser builds a DOM from an HTML document.

CSS: Defines the style. Changes how elements look on the page. Examples include font-size, color, background, position, etc. You can change the style and see it immediately on the page.

Text Editor:A program that that helps you type text similar to document editors; but without all the extra formatting.

  • Sublime: Difficult in the beginning, but has all the languages.
  • Scratchpad: For small projects; beginner-friendly; immediate speed of feedback.
  • CodePen: Beginner-friendly features. Three (3) languages (HTML, CSS, Javascript). Easy to share.

Lesson Three: Adding CSS Style to HTML Structure

Don't Repeat Yourself!!

Use CSS code to add style to your page.

ADD '!DOCTYPE html'

ADD HTML opening and closing tags.

ADD HEAD opening and closing tags.

  • Between the HEAD tags, you add the link to the style file.
  • Start typing, LINK, and then press tab. Sublime automatically adds all the information we need except for the name of the file that contains our style information.

ADD BODY opening and closing tags.

  • Content goes between the BODY tags.
  • For readability, all content between the body tags is indented.

You can add style with a separate file and link to it.

OR

You can add style between opening and closing style tags within the HEAD section.
